音声が4秒程度遅れで聞こえてしまう原因

使用するブラウザやバッファリングにより発生します。

以下の方法で症状が改善する場合があります。

  • レンダリング設定をFlash形式に設定する。Flash形式では、遅延を遅らせるように設計されています。
  • ブラウザにFirefoxかOpus audioを使用する。Opusは次世代で遅延の少ない 低帯域幅のコーデック音声ストリーミング方法であり、Firefoxはバッファリング時間がほとんど発生しないように設計されています。。
  • 視聴ソフトにVLC Media Playerを使用する。 メニューで、 open Media ⇨ Open Network Streamを開き、 http://your-ip:your-port/audio.wavとURLをタイプします。そして "Show more options" をクリックしバッファリングを1 msと設定てください。

タスクコントロール

Tasker とはAndroid端末向けの自動制御アプリです。現在地やスケジュールによってSMSを送信したり、着信音のボリュームを変更したりするような機能のことを指しています。

IP WebcamアプリではウェブインターフェイスやアプリのUIカスタマイズ画面からTaskerイベントの設定が可能です。

IP Webcamアプリでイベントを使用するためにTaskerを設定をするには、「+」ボタンをクリックして、State ⇨ Plugin ⇨ IP Webcamをトリガーとして選択し新しいプロファイルを作成します。

ウェブインターフェイスから作られたイベントはボタンテキストと同じイベント名になります。 例として、中央ボタンは「5」というイベント名になります。

HTTPS使用の場合、警告が出る

DNSを使用しなくてホストにアクセスする場合、HTTPS警告が表示されるのは正常です。

HTTPSを使用すると、安全でないネットワークを使用している場合でも、盗聴者の攻撃から保護されます。

これらの警告は無視しても安全です。
